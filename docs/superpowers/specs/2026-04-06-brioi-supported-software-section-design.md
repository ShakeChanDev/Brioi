# Brioi Supported Software Section Design

**Date:** 2026-04-06  
**Project:** Brioi API 首页局部改版  
**Scope:** 首页 `editorial-blocks` 区块替换为“支持的软件展示”  
**Supersedes:** `/Users/Shake/Documents/App/Brioi/docs/superpowers/specs/2026-04-05-brioi-homepage-design.md` 中原 `Website Introduction Section` 的该区块表达

## 1. Goal

将首页原有的 `Why Brioi API` 三张说明卡，替换为更直观的“支持的软件展示”区块。  
目标不是继续讲平台卖点，而是让用户在首页中段快速看到当前支持的软件范围，并能通过弹窗查看每个软件的使用方式。

该区块必须满足以下要求：

- 保持和现有网站一致的视觉风格，不引入脱节的设计语言
- 以四张软件卡为核心，不再使用说明型文案卡
- 四张卡需要同时展示在一行内
- 每张卡都带可点击的 `使用方式` 按钮
- 点击按钮后打开弹窗，不跳转、不滚动到别处
- 弹窗底部唯一主按钮为 `关闭`

## 2. Content Model

区块固定展示 4 个软件：

1. `Codex`
2. `Claude Code`
3. `OpenCode`
4. `OpenClaw`

每张卡片从上到下的内容顺序固定为：

1. 软件图标
2. 软件名
3. 一行极短辅助信息
4. `使用方式` 按钮

辅助信息用于补足轻度识别，不允许发展成大段说明文案。  
该区块的核心仍是“软件展示”，不是功能说明列表。

## 3. Visual Direction

### 3.1 Overall Style

延续当前首页已经确定的视觉系统：

- 白底
- 黑字
- 绿色作为小面积强调色
- 大圆角卡片
- 高对比、强留白、强排版感

该区块采用“强视觉竖卡”方向，但必须控制在现有站点风格范围内。  
意思是：它要比当前说明卡更有展示感，但不能做成图标商城、应用市场或独立产品页风格。

### 3.2 Card Layout

桌面端为四张竖向卡片并排展示，统一高度，形成清晰的软件展示墙。

每张卡片视觉重点如下：

- 顶部图标区域占据最强识别点
- 中部软件名使用较大字重，保证扫读效率
- 底部按钮保持统一尺寸和位置，形成一致的操作节奏

卡片需要比现有 `info-card` 更有视觉存在感，具体方式包括：

- 更明确的边框或层次
- 更完整的卡片内边距
- 更明显的 hover 态
- 更聚焦的图标展示区

但要避免以下做法：

- 大面积绿色铺底
- 过强的玻璃拟态
- 彩色图标墙导致整页失焦
- 终端窗口壳层或复杂装饰元素

### 3.3 Heading

原 `Why Brioi API / 官方客户端。别买贵的。` 区块标题不再保留。  
新标题应直接表达“支持的软件展示”，语气简洁、直给，与现有首页文案风格一致。

标题结构保持现有 section heading 的语法体系：

- 一行英文小标签
- 一组大号中文标题

标题文案在实现时可微调，但语义必须围绕“支持的软件”而不是“平台优势说明”。

## 4. Interaction Design

### 4.1 Button Behavior

每张卡底部按钮文案固定为 `使用方式`。  
按钮为真实交互元素，不是静态标签。

点击按钮后：

- 打开当前页面中的居中弹窗
- 页面背景出现遮罩
- 当前页面位置不发生跳转
- 不进入新页面
- 不滚动到页面其他区块

### 4.2 Modal Structure

弹窗用于承载该软件的使用方式说明，结构固定为：

1. 软件图标
2. 软件名
3. `适用方式`
4. `使用步骤`
5. `补充说明`

其中：

- `适用方式` 用一句话概括这个软件如何接入或使用
- `使用步骤` 使用短步骤列表呈现
- `补充说明` 只保留必要提示，不扩展成 FAQ

弹窗内容重点是说明“怎么用”，不是再次重复销售话术。

### 4.3 Modal Actions

弹窗底部唯一主按钮固定为 `关闭`。  
不在弹窗底部放购买按钮，也不在此处引导跳转到其他页面。

关闭方式至少包含：

- 底部 `关闭` 按钮

实现时允许额外提供右上角关闭入口，但不是本次设计的核心依赖。

## 5. Responsive Rules

桌面端要求四张卡同排展示，保持“一眼看到全部支持软件”的效果。  
移动端可以改为两列或单列，但仍需保持竖卡结构和清晰的上下层级。

无论在哪个断点下，都应满足：

- 图标始终位于卡片顶部
- 软件名始终在图标下方
- `使用方式` 按钮始终位于卡片底部
- 弹窗在小屏幕下仍能完整阅读和关闭

## 6. Implementation Notes

- 复用现有 section 的外层节奏与版心宽度，避免打破页面整体流速
- 在现有样式系统内新增卡片和弹窗样式，不重写全站设计 token
- 软件图标优先使用可控的本地图形或稳定资源，不依赖不受控的第三方展示接口
- 弹窗交互由现有 `main.js` 承担即可，不引入额外框架

## 7. Success Criteria

当这个区块实现完成后，用户进入首页中段应能立即理解：

1. 当前支持的具体软件有哪些
2. 这四个软件是被正式列出来的，不是口头带过
3. 点击后可以直接看到该软件的使用方式

同时，整块视觉上应被感知为 Brioi 首页的一部分，而不是另外插入的一套 UI。
