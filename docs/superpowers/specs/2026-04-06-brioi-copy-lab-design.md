# Brioi Hero Copy Lab Design

**Date:** 2026-04-06  
**Project:** Brioi API 首页文案对比演示  
**Scope:** 新增独立 `copy-lab` 页面，用于对比首页 Hero 文案版本  
**Supersedes:** 无

## 1. Goal

为首页首屏文案新增一个独立的演示页，让同一套视觉结构下的多个 Hero 文案版本可以直接切换对比。  
目标不是改版正式首页，而是帮助快速判断哪一种标题表达最适合后续首页上线。

这个页面必须满足以下要求：

- 不改动现有首页 `/Users/Shake/Documents/App/Brioi/index.html`
- 不影响当前首页测试断言
- 在同一套 Hero 结构里切换不同文案，而不是做多张不同页面截图
- 文案差异足够清晰，方便直接比较语气与理解成本
- 页面本身看起来仍然像 Brioi 站内页面，而不是临时白板

## 2. Page Structure

演示页新增为独立页面，例如 `/Users/Shake/Documents/App/Brioi/copy-lab.html`。

页面结构从上到下固定为：

1. 顶部导航
2. 页面说明与版本切换器
3. 单个 Hero 预览区
4. 简短版本说明区

其中 Hero 预览区始终只显示一个版本。  
不采用四张 Hero 同屏并排的方式，避免用户被不同布局同时干扰，难以判断真正差异来自文案还是来自版式。

## 3. Visual Direction

### 3.1 Overall Style

演示页沿用当前 Brioi 首页已确认的白底、黑字、绿色强调色视觉系统：

- 保持现有字体、色板、圆角、按钮语言
- Hero 版式尽量复用首页现有结构
- 强调色继续使用现有绿色
- 不额外引入新的插画、装饰背景、渐变主题或营销模块

演示页的重点是“看文案”，不是再做一轮视觉探索。

### 3.2 Hero Preview

Hero 预览区保留当前首页的核心阅读路径：

- 左侧为标题与副标题
- 右侧为套餐卡
- CTA 与已有站点视觉风格保持一致

为了突出文案本身，本次演示中 Hero 的结构、按钮和套餐卡保持稳定。  
切换版本时只替换文案内容，不替换整体排版。

## 4. Copy Variants

演示页固定提供 4 个版本，分别代表不同的语气方向。

### 4.1 Variant A: 原始提案型

主标题：`让更强的大模型 API，被更多人用上`

副标题：`支持最新 GPT-5.4 模型，企业级稳定性，普通用户和团队都能轻松接入。`

### 4.2 Variant B: 更顺口型

主标题：`让更多人用上更强的大模型 API`

副标题：`支持最新 GPT-5.4 模型，稳定接入，普通用户和团队都能直接开用。`

### 4.3 Variant C: 更口语型

主标题：`让好用的大模型 API，服务更多人`

副标题：`支持最新 GPT-5.4 模型，稳定、省心，也更适合长期使用。`

### 4.4 Variant D: 更普惠型

主标题：`让每个人都能轻松用上更强的大模型 API`

副标题：`支持最新 GPT-5.4 模型，把更强能力用更简单的方式带给更多人。`

所有版本中：

- `GPT-5.4` 必须使用强调色
- 文案切换时，除标题与副标题外，其他 Hero 元素尽量保持不变

## 5. Interaction Design

### 5.1 Variant Switcher

页面顶部提供 4 个可点击的版本切换按钮，对应 Variant A-D。

交互要求：

- 默认展示 Variant A
- 点击任一版本按钮后，Hero 预览区立即切换到对应文案
- 当前选中的版本按钮需要有明显的选中态
- 用户可以来回切换，不发生整页跳转

### 5.2 Copy Context

在 Hero 预览区下方或旁边，提供一段极简说明，标记当前版本代表的语气方向，例如：

- 原始提案型
- 更顺口型
- 更口语型
- 更普惠型

说明的目的是帮助判断“哪一类语气更适合首页”，不是写成长解释。

## 6. Implementation Notes

- 通过新页面承载演示，不修改现有首页 Hero 正式文案
- 复用现有样式系统，新增最少量的 `copy-lab` 专属样式
- 在现有 `/Users/Shake/Documents/App/Brioi/main.js` 中新增仅对 `data-page="copy-lab"` 生效的初始化逻辑
- 文案版本数据采用前端静态配置即可，不引入额外依赖
- 允许复用现有 Hero 套餐卡结构，避免重复造样式

## 7. Test Expectations

至少需要覆盖以下验证点：

1. 演示页可以正常打开
2. 默认显示 Variant A
3. 点击切换按钮后，Hero 主标题与副标题正确更新
4. `GPT-5.4` 在副标题中以强调样式呈现
5. 切换版本不会影响现有首页测试

## 8. Success Criteria

当这个演示页完成后，应满足以下结果：

1. 用户可以在同一页内快速比较 4 个首页 Hero 文案方向
2. 可以清楚判断哪一句最顺、最像人话、最适合首页首屏
3. 当前正式首页保持不变，不产生回归风险
