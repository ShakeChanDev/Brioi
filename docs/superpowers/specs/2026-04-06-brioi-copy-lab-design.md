# Brioi Hero Copy Lab Design

**Date:** 2026-04-06  
**Project:** Brioi API 首页文案对比演示  
**Scope:** 重设计独立 `copy-lab` 页面，解决长中文 Hero 标题排版失衡问题  
**Supersedes:** 本文件此前版本中“单层文案切换 + 直接复用首页 Hero 排版”的设计思路

## 1. Goal

保留独立 `copy-lab` 演示页方向，但重做首屏预览的排版系统。  
这次的核心不是继续增加文案版本，而是解决一个已经确认的根问题：现有超大 Hero 标题排版不适合这组较长的中文文案，导致字面内容本身被排版拖累，难以判断哪句文案真正更好。

这个页面必须满足以下要求：

- 不改动现有首页 `/Users/Shake/Documents/App/Brioi/index.html`
- 不影响当前首页测试断言
- 继续保留 4 个文案版本供切换比较
- 新增 3 个排版模式，让用户能直接判断“是文案问题还是排版问题”
- Hero 预览的阅读焦点必须回到文案本身，而不是页面说明或右侧套餐卡
- 页面看起来仍然属于 Brioi 站内，而不是单纯的内部工具页

## 2. Problem Diagnosis

当前版本的问题已经收敛为以下几点，其中最主要的是第一条：

1. 长中文标题直接套用了首页的超大号 Hero 标题系统，导致自动换行结果僵硬、行长失衡、阅读节奏破碎。
2. 标题断句依赖浏览器自动换行，而不是人为控制，因此每个版本都像“被撑开”的临时排版。
3. 页面顶部说明过重，抢走了 Hero 本来应作为首要阅读入口的视觉焦点。
4. 右侧套餐卡视觉重量过大，影响用户专注比较文案。

本次重设计只围绕这些问题展开，不扩展到新的营销模块，也不再发散视觉主题。

## 3. Page Structure

演示页仍使用独立页面，例如 `/Users/Shake/Documents/App/Brioi/copy-lab.html`。

页面结构从上到下固定为：

1. 顶部导航
2. 极简页面说明
3. 双层切换器
4. 单个 Hero 预览区
5. 当前模式与当前文案的极简标记区

其中：

- 页面说明只保留一行极短介绍，不再使用大号标题抢视觉
- Hero 预览区始终只显示一个版本
- 页面不做多宫格对比，不做多个首屏并排

原因很简单：这次是排版系统比较，不是页面布局比较。单预览区更容易看出真正差异。

## 4. Interaction Model

### 4.1 Copy Variant Switcher

保留 4 个文案版本切换：

1. Variant A：`让更强的大模型 API，被更多人用上`
2. Variant B：`让更多人用上更强的大模型 API`
3. Variant C：`让好用的大模型 API，服务更多人`
4. Variant D：`让每个人都能轻松用上更强的大模型 API`

副标题继续固定围绕 `支持最新 GPT-5.4 模型` 方向展开，并在切换版本时同步更新。

### 4.2 Typography Mode Switcher

新增一组并列的排版模式切换：

1. `整齐多行型`
2. `官网稳重型`
3. `海报断句型`

这组切换器和文案切换器共同决定 Hero 最终呈现。  
也就是说，用户不是只在看“哪一句更好”，而是在看“哪一句在不同排版里表现如何”。

### 4.3 Combined Behavior

页面必须支持两层切换同时存在：

- 第一层：决定文案内容
- 第二层：决定该文案如何断句、字号如何分配、行数如何控制

用户点击任一层的按钮后：

- Hero 标题立即更新
- 当前选中按钮具有明确选中态
- 页面不跳转
- 不刷新整个页面

## 5. Typography Strategy

这次的核心设计不是“换个字号”，而是把长中文标题从自动换行改成“逐行排版”。

### 5.1 Line-Based Rendering

Hero 主标题不再作为一整段文字直接交给浏览器换行。  
每个文案版本在每个排版模式下，都需要有明确的逐行方案，例如：

- 哪些词在第一行
- `API` 是否单独成行
- 哪一行更长，哪一行更短
- 哪一行承担强调作用

实现时主标题应按“多行单元”渲染，而不是单纯依赖单个文本节点自动折行。

### 5.2 Mode A: 整齐多行型

这个模式的目标是“最顺、最易读”。

要求：

- 标题仍保持较强视觉冲击，但行长尽量均衡
- 每行字数控制更整齐
- 行距略紧，避免长标题显得松散
- 优先保证第一眼扫读顺畅，而不是追求极端视觉张力

适合判断：

- 这句文案本身是否通顺
- 同一批文案里谁最适合正式首页

### 5.3 Mode B: 官网稳重型

这个模式的目标是“更像成熟产品官网”。

要求：

- 标题字号收一档
- 行宽更稳，留白更克制
- 副标题与主标题的层级关系更清楚
- 整体更偏正式首页，而不是海报

适合判断：

- 哪一句文案在正式上线时最稳
- 哪一句即使不靠极大字体也能成立

### 5.4 Mode C: 海报断句型

这个模式的目标是“保留冲击力，但断句更有意图”。

要求：

- 可以有更多行数
- 允许某些关键词独立成行
- 视觉上更有海报节奏感
- 不能退化成随机换行或破碎堆字

适合判断：

- 哪一句文案更有传播感
- 哪一句在强节奏排版下仍然成立

## 6. Visual Direction

### 6.1 Overall Style

沿用当前 Brioi 的白底、黑字、绿色强调色系统，不改主题。

要求：

- 保持现有字体与色板
- 不新增插画、背景装饰或复杂光效
- 页面视觉语言继续属于 Brioi

### 6.2 Intro Area

页面顶部说明必须降权：

- 只保留一行极简说明
- 不再出现一个比 Hero 更醒目的说明大标题
- 用户打开页面后第一眼应落在 Hero 文案，而不是工具说明

### 6.3 Hero Preview

Hero 预览区依然保留“左文案、右套餐卡”的真实首页关系，但视觉权重必须调整：

- 左侧文案区扩大阅读优先级
- 右侧套餐卡缩小或减弱视觉对比
- 套餐卡保留的唯一目的，是让你在接近真实首页的条件下比较文案，而不是再次成为页面主角

### 6.4 Subtitle

副标题不能再作为一段随意说明文字存在。  
需要同步进入排版系统：

- 控制最大宽度
- 控制字重与行高
- 确保它是主标题的“解释层”，而不是额外噪音

其中 `GPT-5.4` 必须继续使用强调色。

## 7. Content Model

### 7.1 Copy Data

需要有一份独立的文案数据源，至少包含：

- `title`
- `subtitle`
- `note`

### 7.2 Typography Data

需要有一份独立的排版模式数据源，至少包含：

- 模式名
- 标题字号级别
- 行距级别
- 套餐卡权重策略

### 7.3 Line Break Mapping

需要有一份“文案版本 × 排版模式”的断句映射。  
也就是每个文案版本在每种模式下，都有自己明确的分行结果。

这样做的目的，是让后续调整文案和调整排版可以分离进行，而不是互相绑死。

## 8. Implementation Notes

- 继续通过 `copy-lab` 独立页面承载演示
- 不修改现有首页正式文案
- 保留现有 `main.js` 页面级初始化思路，但扩展为双层切换器
- 标题渲染应支持多行结构输出
- 新样式应专注于 copy-lab 页面，不影响现有首页和购买页

## 9. Test Expectations

至少需要覆盖以下验证点：

1. 演示页可以正常打开
2. 默认文案版本与默认排版模式都正确生效
3. 切换文案版本后，主标题与副标题正确更新
4. 切换排版模式后，标题的逐行结构发生预期变化
5. copy-lab 中的 Hero 文案在有 JS 时仍可见，不会再次被首页 reveal 逻辑隐藏
6. `GPT-5.4` 在副标题中仍以强调样式呈现
7. 现有首页测试不受影响

## 10. Success Criteria

当这个新版演示页完成后，应满足以下结果：

1. 用户可以明确区分“文案本身”和“排版方式”的影响
2. 长中文标题不再显得被硬拆、被撑裂或随机换行
3. 同一组文案在三种模式下都有可比较的正式呈现
4. 页面第一焦点回到 Hero 文案，而不是说明区或价格卡
