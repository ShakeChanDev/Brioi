# Brioi Hero Copy Lab Design

**Date:** 2026-04-06  
**Project:** Brioi API 首页文案对比演示  
**Scope:** 重设计独立 `copy-lab` 页面，针对一组固定两行标题与一条副标题比较 3 种排版模式  
**Supersedes:** 本文件此前版本中“文案版本切换 + 排版模式切换”的设计思路

## 1. Goal

保留独立 `copy-lab` 演示页方向，但把比较目标进一步收窄。  
这次不再比较多句不同文案，而是固定一组已经确认的两行标题与一条副标题，只比较排版方式本身。这样可以排除“句子不同”带来的干扰，直接判断长中文标题该怎样排才最顺、最稳、最适合首页。

这个页面必须满足以下要求：

- 不改动现有首页 `/Users/Shake/Documents/App/Brioi/index.html`
- 不影响当前首页测试断言
- 固定展示一组标题与副标题，不再保留 `A/B/C/D` 文案版本切换
- 保留 3 个排版模式，让用户直接判断“同一句文案该怎么排”
- Hero 预览的阅读焦点必须回到文案本身，而不是页面说明或右侧套餐卡
- 页面看起来仍然属于 Brioi 站内，而不是单纯的内部工具页

## 2. Problem Diagnosis

当前版本的问题已经收敛为以下几点，其中最主要的是第一条：

1. 长中文标题直接套用了首页的超大号 Hero 标题系统，导致自动换行结果僵硬、行长失衡、阅读节奏破碎。
2. 标题断句依赖浏览器自动换行，而不是人为控制，因此同一句文案在视觉上像“被撑开”的临时排版。
3. 页面顶部说明过重，抢走了 Hero 本来应作为首要阅读入口的视觉焦点。
4. 右侧套餐卡视觉重量过大，影响用户专注比较文案。

本次重设计只围绕这些问题展开，不扩展到新的营销模块，也不再发散视觉主题。

## 3. Page Structure

演示页仍使用独立页面，例如 `/Users/Shake/Documents/App/Brioi/copy-lab.html`。

页面结构从上到下固定为：

1. 顶部导航
2. 极简页面说明
3. 单层排版模式切换器
4. 单个 Hero 预览区
5. 当前模式的极简标记区

其中：

- 页面说明只保留一行极短介绍，不再使用大号标题抢视觉
- Hero 预览区始终只显示一个版本
- 页面不做多宫格对比，不做多个首屏并排

原因很简单：这次是固定文案下的排版系统比较，不是页面布局比较，也不是文案版本比较。单预览区更容易看出真正差异。

## 4. Interaction Model

### 4.1 Fixed Copy

本轮演示页固定使用以下主标题与副标题：

主标题：

1. `让每个人都能轻松用上`
2. `更强的 AI 大模型`

副标题：

`支持最新 GPT-5.4 模型`

要求：

- 主标题固定为两行，不再作为多句文案切换
- 副标题固定为一行，不再带额外解释句
- `GPT-5.4` 继续使用强调色

### 4.2 Typography Mode Switcher

新增一组并列的排版模式切换：

1. `整齐多行型`
2. `官网稳重型`
3. `海报断句型`

用户点击任一模式按钮后：

- Hero 标题立即更新
- 当前选中按钮具有明确选中态
- 页面不跳转
- 不刷新整个页面

## 5. Typography Strategy

这次的核心设计不是“换个字号”，而是把固定两行标题从自动换行改成“有意图的逐行排版”，并同时抬高副标题层级。

### 5.1 Line-Based Rendering

Hero 主标题不再作为一整段文字直接交给浏览器换行。  
实现上仍应按逐行单元渲染，但这次每种模式都必须围绕固定的两行标题来组织：

1. `让每个人都能轻松用上`
2. `更强的 AI 大模型`

也就是说：

- 两行文本内容固定
- 模式之间主要变化的是字号、行距、块间距、整体节奏
- 不再靠切换不同文案来制造比较

### 5.2 Mode A: 整齐多行型

这个模式的目标是“最顺、最易读”。

要求：

- 标题仍保持较强视觉冲击，但两行宽度要尽量均衡
- 第一行和第二行之间的节奏应稳定、顺滑
- 行距略紧，避免长标题显得松散
- 优先保证第一眼扫读顺畅，而不是追求极端视觉张力

适合判断：

- 这组固定两行标题在最自然状态下是否成立
- 首页主标题最顺的一种呈现是什么

### 5.3 Mode B: 官网稳重型

这个模式的目标是“更像成熟产品官网”。

要求：

- 标题字号收一档
- 行宽更稳，留白更克制
- 副标题与主标题的层级关系更清楚
- 整体更偏正式首页，而不是海报

适合判断：

- 这组固定标题在正式产品首页里是否足够稳
- 不靠极大字号时，它是否仍然成立

### 5.4 Mode C: 海报断句型

这个模式的目标是“保留冲击力，但断句更有意图”。

要求：

- 仍然保持固定两行标题，不额外拆成第三行
- 通过字号比、留白和对齐关系制造更强节奏
- 视觉上更有海报节奏感
- 不能退化成随机换行或破碎堆字

适合判断：

- 这组固定标题在更强张力下是否更有传播感
- 同样两行文本在强节奏排版下是否仍然成立

## 6. Visual Direction

### 6.1 Overall Style

沿用当前 Brioi 的白底、黑字、绿色强调色系统，不改主题。

要求：

- 保持现有字体与色板
- 不新增插画、背景装饰或复杂光效
- 页面视觉语言继续属于 Brioi

### 6.2 Intro Area

页面顶部说明必须降权：

- 只保留一行极简说明
- 不再出现一个比 Hero 更醒目的说明大标题
- 用户打开页面后第一眼应落在 Hero 文案，而不是工具说明

### 6.3 Hero Preview

Hero 预览区依然保留“左文案、右套餐卡”的真实首页关系，但视觉权重必须调整：

- 左侧文案区扩大阅读优先级
- 右侧套餐卡缩小或减弱视觉对比
- 套餐卡保留的唯一目的，是让你在接近真实首页的条件下比较文案，而不是再次成为页面主角

### 6.4 Subtitle

副标题不能再作为一段随意说明文字存在。  
这次副标题固定为 `支持最新 GPT-5.4 模型`，并且层级要上调：

- 字号必须明显小于主标题
- 但必须明显大于当前演示页里那种说明文字大小
- 它应更接近“次级标题”而不是“正文说明”
- 仍需控制字重与行高，确保它是主标题的解释层，而不是额外噪音

其中 `GPT-5.4` 必须继续使用强调色。

## 7. Content Model

### 7.1 Fixed Copy Data

需要有一份固定文案数据源，至少包含：

- `titleLines`
- `subtitle`

### 7.2 Typography Data

需要有一份独立的排版模式数据源，至少包含：

- 模式名
- 标题字号级别
- 行距级别
- 套餐卡权重策略

### 7.3 Line Break Mapping

需要有一份“固定标题 × 排版模式”的呈现映射。  
也就是同样的两行标题，在每种模式下都可以有自己的字号、行距、块间距和对齐策略。

这样做的目的，是让固定文案与排版模式解耦，后续如果只想继续打磨样式，不需要再引入新的文案变量。

## 8. Implementation Notes

- 继续通过 `copy-lab` 独立页面承载演示
- 不修改现有首页正式文案
- 保留现有 `main.js` 页面级初始化思路，但切换器只保留排版模式这一层
- 标题渲染应支持多行结构输出
- 新样式应专注于 copy-lab 页面，不影响现有首页和购买页

## 9. Test Expectations

至少需要覆盖以下验证点：

1. 演示页可以正常打开
2. 默认两行标题与默认排版模式都正确生效
3. 切换排版模式后，标题与副标题的排版层级发生预期变化
4. 标题始终保持两行固定内容，不再出现额外文案切换
5. copy-lab 中的 Hero 文案在有 JS 时仍可见，不会再次被首页 reveal 逻辑隐藏
6. `GPT-5.4` 在副标题中仍以强调样式呈现
7. 现有首页测试不受影响

## 10. Success Criteria

当这个新版演示页完成后，应满足以下结果：

1. 用户看到的是同一组固定文案在 3 种模式下的直接对比，而不是被多组文案干扰
2. 两行标题不再显得被硬拆、被撑裂或随机换行
3. 副标题层级明显提升，更像次级标题而不是说明文字
4. 页面第一焦点回到 Hero 文案，而不是说明区或价格卡
