# Brioi Runtime 22222 Design

**Date:** 2026-04-06  
**Project:** Brioi API 运行时配置收口  
**Scope:** 删除本地开发命名入口，只保留线上启动入口，并将默认端口统一为 `22222`

## 1. Goal

把当前项目的启动配置从“开发环境命名”收成“线上环境命名”：

- `package.json` 不再保留 `dev` 启动脚本
- 只保留一个面向线上环境的启动入口
- 默认端口统一改为 `22222`
- 测试配置继续保留，但要跟随新的启动命令和端口
- 改完后实际启动服务并打开页面

## 2. Current State

当前运行时配置集中在两个文件：

1. `/Users/Shake/Documents/App/Brioi/package.json`
2. `/Users/Shake/Documents/App/Brioi/playwright.config.js`

现状是：

- `package.json` 使用 `dev` 脚本启动 `http-server`
- 默认端口为 `4173`
- Playwright 也固定依赖 `4173` 和 `npm run dev`

这和“只保留线上环境”的目标不一致，因为运行入口仍然是开发环境命名。

## 3. Planned Changes

### 3.1 package.json

将脚本改成单一线上入口：

- 删除 `dev`
- 新增或保留 `start`
- `start` 命令改为 `http-server . -p 22222 -c-1`

`test` 脚本保持不变。

### 3.2 playwright.config.js

测试配置继续保留，但同步新的运行入口：

- `use.baseURL` 改为 `http://127.0.0.1:22222`
- `webServer.command` 改为 `npm run start`
- `webServer.port` 改为 `22222`

不删除 Playwright，也不把它算进“dev 相关配置”范围。

### 3.3 Non-Goals

以下内容不在本次范围内：

- 不改首页文案或页面结构
- 不替换 `http-server`
- 不新增环境变量系统
- 不清理测试体系
- 不改业务代码

## 4. Verification

完成后需要验证：

1. `npm test` 可以在新端口配置下通过
2. `npm run start` 能默认在 `22222` 端口启动
3. 本地浏览器可打开 `http://127.0.0.1:22222`

## 5. Risks

本仓库当前主工作区有未提交的首页改动，因此本次实现放在独立 worktree 中进行，避免把运行时配置修改与现有页面改动混在一起。
