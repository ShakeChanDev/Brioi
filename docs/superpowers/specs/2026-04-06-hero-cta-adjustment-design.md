# Brioi Hero CTA Adjustment Design

**Date:** 2026-04-06  
**Project:** Brioi API 首页 CTA 微调  
**Scope:** 首页 Hero 区域按钮重排

## 1. Goal

调整首页 Hero 区域的按钮布局，减少左侧文案区的重复 CTA，并把主操作集中到右侧套餐卡片下方。

本次改动只处理 Hero 首屏 CTA，不修改定价区、页尾 CTA、购买页逻辑或其他文案结构。

## 2. Confirmed Requirements

- 删除左侧文案区当前两个按钮：
  - `购买 Plus 月卡`
  - `查看套餐`
- 将右侧 `Plus 月卡` 卡片底部当前单个按钮替换为两个并排按钮
- 左侧新按钮文案为 `去购买`
- 右侧新按钮文案为 `查看更多套餐`
- 两个新按钮都跳转到当前 `查看套餐` 的目标位置，即 `#pricing`

## 3. Interaction Rules

- Hero 左侧文案区不再保留 CTA 容器占位
- Hero 右侧卡片成为首屏唯一的套餐操作入口
- 桌面端两个按钮横向并排展示
- 小屏端允许改为纵向堆叠，以避免按钮宽度不足或文本挤压

## 4. Visual Direction

- 延续当前首页已存在的 Hero 视觉系统
- 复用现有按钮 token，不单独引入新的色板或交互模式
- 在右侧深色卡片内保持主次分明：
  - `去购买` 使用当前更强的主按钮表现
  - `查看更多套餐` 使用次级按钮表现，但仍需保证在深色卡片上可读

## 5. Success Criteria

实现完成后，Hero 区域应满足：

1. 左侧不再出现重复 CTA
2. 右侧卡片下方清晰展示两个操作按钮
3. 两个按钮均可滚动到 `Pricing` 区块
4. 桌面和移动端都不出现按钮拥挤、溢出或难以点击的问题
